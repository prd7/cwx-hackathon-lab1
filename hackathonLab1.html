Overview

This lab guides developers through building a multilingual support agent in Salesforce using Agentforce, Topics and Actions, and Google Gemini API via Apex.

Pre-requisites

  • Salesforce Developer Org with Einstein Agentforce enabled.
  • Access to setup Named Credentials or Remote Site Settings (we'll use Remote Site for simplicity).
  • A Google Gemini API key (provided during hackathon or use your own).
  • Basic familiarity with Apex, Flow Builder, and Setup.

Step 1: Configure Remote Site

  1. Go to Setup > Remote Site Settings.
  2. Create a new entry:
    • Name: GeminiAPI
    • URL: https://generativelanguage.googleapis.com
    • Active: Yes

Step 2: Create Apex Class

Create the following Apex class called MultiLingualSupportController:

public with sharing class MultiLingualSupportController {
  // ... (Your Apex code here) ...
}

Step 3: Build a Flow

  1. Go to Setup > Flow, create a new flow.
  2. Add an Action element.
  3. Select MultiLingualSupportController.getMultilingualReplyFromFlow.
  4. Pass the input message and (optionally) order context.
  5. Capture the replyText in a Text variable.
  6. Add a screen element to show the result.
  7. Save and activate the flow.

Step 4: Configure Agentforce

  1. Go to Setup > Agentforce.
  2. Create a new agent: Multilingual Support Agent.
  3. Create a topic: Multilingual Inquiry Handling.
  4. Use this topic prompt: Use this topic to process customer questions in various languages and provide appropriate responses.
  5. Let the system generate language-handling logic, or refine it.

Step 5: Add an Agent Action

  1. Under your topic, add a new action of type Flow.
  2. Select your flow from the dropdown.
  3. Action Instructions: "Send the user's message to the multilingual assistant and return its reply."
  4. Loading Text: "Generating multilingual response..."
